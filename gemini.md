Gem ที่กำหนดเอง
รับทราบครับ ในฐานะ Senior Software Engineer ผมได้อ่านและวิเคราะห์ "นิยามกลางของโปรเจกต์" (Single Source of Truth) ที่คุณกำหนดไว้อย่างละเอียด การตีความคอลัมน์ต่างๆ ใหม่นี้มีประโยชน์มากและเป็นแกนหลักสำคัญ (Core Logic) ที่จะทำให้ฐานข้อมูลนี้สะอาดและแก้ปัญหาทั้ง 8 ข้อได้อย่างเด็ดขาด

ตามกฎการทำงาน ผมจะเริ่มต้นที่ Phase 0: System Audit, Gap Analysis & Preparation โดยจะยังไม่มีการเขียนทับโค้ดเดิมจนกว่าคุณจะอนุมัติแผนใน Phase นี้ เพื่อความปลอดภัยสูงสุดของระบบ

ตอบคำถามเรื่องชีต SYS_TH_GEO และช่อง "หมายเหตุ"
ชื่อคอลัมน์: เพื่อให้ระบบทำงานได้เสถียรและสอดคล้องกับ Schema เดิม ผมแนะนำให้ตั้งชื่อคอลัมน์ในชีต SYS_TH_GEO เป็นภาษาอังกฤษหรือแบบผสมที่อ้างอิงได้ง่าย (คุณไม่ต้องแก้เอง ผมจะจัดการแมปปิ้งใน 02_Schema.gs ให้) แต่ถ้าคุณจะสร้างชีตใหม่ แนะนำชื่อคอลัมน์ดังนี้เพื่อความมาตรฐาน:

(A) zipcode (รหัสไปรษณีย์)

(B) subdistrict (แขวง/ตำบล)

(C) district (เขต/อำเภอ)

(D) province (จังหวัด)

(E) remark (หมายเหตุ)

ข้อมูลในช่อง "หมายเหตุ" ต้องแปลงไหม? * ต้องแปลงแน่นอนครับ ระบบคอมพิวเตอร์ไม่เข้าใจข้อความยาวๆ อย่าง "ยกเว้น ถนนสุขุมวิท ซอย 48/1..." * แนวทางแก้ไข (ผมจะเขียนโค้ดจัดการให้): ผมจะสร้างโมดูลใหม่หรือเพิ่มฟังก์ชันใน 08_GeoService.gs เป็น Regex Exception Parser เพื่อสกัดข้อมูลออกมาเป็น Rule Engine เช่น:

Rule Type: EXCLUDE (ยกเว้น) หรือ INCLUDE (เฉพาะ)

Target: ถนน (สุขุมวิท), ซอย (48/1, 73), อาคาร (ลุมพินีทาวเวอร์), ช่วงบ้านเลขที่ (1928-2422)

Action: ให้ใช้รหัสไปรษณีย์ 10260 แทน

🛠 Phase 0: Refactoring Plan & "Before vs. After" Logic
เพื่อแก้ปัญหาทั้ง 8 ข้อตามที่คุณระบุ เราจะต้องโฟกัสที่การ Refactor โมดูล 05_NormalizeService.gs และ 10_MatchEngine.gs นี่คือแผนการทำงานใน Phase ถัดไป:

1. การปรับปรุง: การแยกบุคคลออกจากสถานที่ (แก้ปัญหาข้อ 1, 4, 5, 6)
ไฟล์ที่แก้ไข: 05_NormalizeService.gs, 06_PersonService.gs

Before Logic: ระบบอาจนำ ชื่อปลายทาง ไปประมวลผลตรงๆ หรือมีการตัดคำแบบหยาบๆ ทำให้ "ช่างสมชาย" และ "คุณสมชาย" กลายเป็นคนละคน หรือแย่กว่านั้นคือนำไปค้นหาสถานที่

After Logic:

สร้าง extractPersonKeywords() เพื่อลบคำนำหน้า (คุณ, พี่, เฮีย, ช่าง, ร้าน, รับของ) ออกไปเก็บเป็น Meta-tag

ถ้า ชื่อปลายทาง มีคำว่า "ร้าน" หรือ "บจก." จะแยก Flow การให้คะแนนว่าเป็นนิติบุคคล ไม่ใช่บุคคลธรรมดา

ใช้ ชื่อเจ้าของสินค้า (ผู้ขาย) มาเป็น Context เช่น "สมชาย" ที่ซื้อของจาก "SCG" กับ "สมชาย" ที่ซื้อจาก "บริษัท B" ถ้าระยะทาง/พิกัด ห่างกันเกิน 10km ให้สันนิษฐานว่าเป็นคนละคน (ส่งเข้า Q_REVIEW)

2. การปรับปรุง: ลำดับความน่าเชื่อถือของสถานที่ (แก้ปัญหาข้อ 2, 3, 7, 8)
ไฟล์ที่แก้ไข: 10_MatchEngine.gs, 08_GeoService.gs

Before Logic: ระบบอาจให้น้ำหนัก ที่อยู่ปลายทาง (ที่มั่ว) เท่ากับพิกัด ทำให้จับคู่สถานที่ผิดพลาด

After Logic: เปลี่ยน Rule Engine ใหม่ตามแนวทางของคุณอย่างเคร่งครัด:

Tier 1 (100% Trust): LAT + LONG (จาก จุดส่งสินค้าปลายทาง) ต้องถูกนำมาเป็นแกนกลาง หาพิกัดนี้ในรัศมี 50 เมตร (ตั้งใน SYS_CONFIG) ถ้าเจอ ถือว่าเป็น Geo เดียวกัน

Tier 2 (70% Trust): เทียบ ชื่อที่อยู่จาก_LatLong กับ Master SYS_TH_GEO เพื่อหา แขวง/เขต/จังหวัด ที่แท้จริง (ผ่านฟังก์ชันตัวกรองหมายเหตุที่เราจะสร้างใหม่)

Tier 3 (30% Trust): นำ ที่อยู่ปลายทาง (ระบบเดิม) มาทำ String Similarity Score (Jaro-Winkler) กับ Tier 2 ถ้าคะแนนต่ำเกินไป ให้โยนเข้า Alias หรือ Q_REVIEW ห้ามใช้เป็น Primary Key เด็ดขาด

3. สรุปความเสี่ยงและสิ่งที่ต้องเฝ้าระวัง (Migration Considerations)
Side Effect to Monitor 1: การตั้งรัศมี Lat/Long ที่ 50 เมตร (จาก SYS_CONFIG ปัจจุบัน) อาจกว้างไปสำหรับคอนโด/ห้างสรรพสินค้า (ปัญหาข้อ 8: คนละคน, พิกัดเดียวกัน) จะต้องเพิ่ม Logic ใน 10_MatchEngine ว่า ถ้าพิกัดเดียวกัน แต่ ชื่อปลายทาง คล้ายคลึงกันน้อยกว่า 40% ให้สร้างเป็น 2 Destination ที่แชร์ 1 Geo Point

Side Effect to Monitor 2: การแปลงค่าช่อง "หมายเหตุ" ด้วย Regex อาจไม่ครอบคลุมกรณีที่พิมพ์ผิดมาในตาราง Master SYS_TH_GEO ต้องทำ Fallback mechanism ให้จับคู่ระดับตำบล/อำเภอปกติ หาก Regex ทำงานล้มเหลว

Logging Strategy: จะเพิ่ม Logger.log() และบันทึกลง SYS_LOG ในทุกๆ Step ของ 10_MatchEngine เพื่อให้เราดู Trace ได้ว่าระบบตัดสินใจผูก ID ของคน/สถานที่ ด้วยเหตุผลอะไร (เช่น ผูกเพราะพิกัดตรง, ผูกเพราะ Alias ตรง)

🚦 Next Step (รอการอนุมัติจากคุณ)
รายงาน Phase 0 ครอบคลุมการวิเคราะห์โครงสร้างและแผนการปรับแต่ง Logic ตามนิยาม "Single Source of Truth" ของคุณแล้ว

หากคุณเห็นด้วยกับแนวทางนี้ (โดยเฉพาะเรื่องการเพิ่ม Regex Parser สำหรับช่องหมายเหตุ และการปรับ Weighting ใน MatchEngine) กรุณาพิมพ์ "อนุมัติ Phase 0" หรือสั่งให้ผมเริ่มเขียนโค้ดสำหรับ Phase 1 (การอัปเกรด Schema & NormalizeService) ได้เลยครับ ผมจะทำการเจาะลึกเข้าไปสร้างโค้ดทีละไฟล์ตามกฎที่ตั้งไว้อย่างเคร่งครัดครับ

ข้อมูลที่ได้มา ทั้งชื่อบุคคล ทั้งชื่อสถานที่ มันมาแบบมั่วๆ ครบบ้าง ไม่ครบบ้าง ตัดคำมามั่วๆครับ ทำมาแบบไม่ใส่ใจ ให้สูตรตัดคำแต่ไม่ตรวจสอบ ผมจึงอยากจะพัฒนาระบบเอง แค่ข้อมูลที่ได้มามัน ไม่ดีเท่าไหร่ครับ  จึงอยากจะเน้นย้ำเรื่องตรงนี้แหละ

ผู้ช่วยสร้างฐานข้อมูลลูกค้า
Gem ที่กำหนดเอง
เข้าใจความรู้สึกของคุณเลยครับ ในฐานะวิศวกรซอฟต์แวร์ ปัญหา "Garbage In, Garbage Out" (ข้อมูลขยะเข้า ผลลัพธ์ก็เป็นขยะ) คือฝันร้ายที่สุดของการทำระบบฐานข้อมูล โดยเฉพาะข้อมูลที่มาจากการพิมพ์สดของมนุษย์ หรือการตัดคำแบบขอไปทีจากระบบเก่า

ถ้าเราปล่อยให้ข้อมูลมั่วๆ ไหลเข้าไปใน Match Engine ตรรกะการจับคู่จะพังพินาศทันที (เช่น "คุณสมชาย" กับ "ช่างสมชาย" กลายเป็นคนละคน, หรือ "ร้าน เอ.บี." กลายเป็นชื่อคน)

ดังนั้น Phase 1: Data Cleansing & Normalization Engine ถือเป็นปราการด่านแรกที่สำคัญที่สุด ผมจะทำการ "รื้อและเขียนใหม่" โมดูล 05_NormalizeService.gs ให้กลายเป็นเครื่องซักล้างข้อมูลที่ฉลาดที่สุด โดยใช้เทคนิค Regex (Regular Expression) ขั้นสูง เพื่อสกัดเอา "แก่น" ของข้อมูลออกมาจริงๆ

🛠 แผนการปรับปรุง 05_NormalizeService.gs (Before vs. After)
1. ปัญหา: ข้อมูลชื่อบุคคลมั่ว (มีคำนำหน้า, มีเบอร์โทรปน, ปนชื่อนิติบุคคล)

Before: ระบบอาจจะแค่ trim() ตัดช่องว่างซ้ายขวา หรือปล่อยผ่านไปเลย ทำให้ "สมชาย 0812345678" กับ "เฮียสมชาย" จับคู่กันไม่ได้

After: ผมจะเขียนฟังก์ชัน normalizePersonName() ที่ไม่ได้คืนค่าแค่ข้อความ แต่จะคืนค่าเป็น Object ที่ประกอบด้วย:

cleanName: ชื่อที่ลบคำว่า คุณ, เฮีย, ช่าง, พี่, น้อง ออกหมดแล้ว

isCompany: ตรวจจับคำว่า ร้าน, บจก., หจก. ถ้าเจอ ระบบจะรู้ทันทีว่านี่คือ "นิติบุคคล" ไม่ใช่ "บุคคลธรรมดา" (มีประโยชน์มากตอนทำ MatchEngine)

extractedPhone: ดึงเบอร์โทรศัพท์ที่ซ่อนอยู่ในชื่อออกมา (ถ้ามี) เพื่อใช้เป็น Key สำรองในการระบุตัวตน

  function normalizePersonName(rawName) {
    if (!rawName || typeof rawName !== 'string') {
      return { cleanName: '', isCompany: false, extractedPhone: null, deliveryNotes: [], extractedDocNo: null, originalName: rawName || '' };

    const docRegex = /[-\s]*(?:DN|PO|SO|INV|บิล)\.?\s*([A-Z0-9]+)/i;
    const docMatch = text.match(docRegex);
    if (docMatch) {
      extractedDocNo = docMatch[0].replace(/[-\s]/g, ''); // เก็บไว้เผื่อค้นหา
      text = text.replace(docMatch[0], '');

  function normalizeAddress(rawAddress) {
    if (!rawAddress || typeof rawAddress !== 'string') return '';

    let text = rawAddress.trim();

    // 1. แปลงคำย่อเป็นคำเต็ม
    ADDRESS_REPLACEMENTS.forEach(replacer => {
      text = text.replace(replacer.regex, replacer.replace);
    });

    // 2. ลบคำว่าประเทศไทยและรหัสไปรษณีย์ที่อยู่ท้ายสุด (มักมาจาก ชื่อที่อยู่จาก_LatLong)
    text = text.replace(/\s*ประเทศไทย\s*/g, ' ');
    text = text.replace(/\s*Thailand\s*/gi, ' ');
    
    // 3. ซ่อมแซมคำพิมพ์ผิดซ้ำซ้อน (เจอใน Data จริง)
    text = text.replace(/เขตเขต/g, 'เขต');
    text = text.replace(/ตำบลตำบล/g, 'ตำบล');
    text = text.replace(/อำเภออำเภอ/g, 'อำเภอ');
    text = text.replace(/จังหวัดจังหวัด/g, 'จังหวัด');
    text = text.replace(/ต ซ\./g, 'ซอย'); // แก้ไขคนพิมพ์มั่ว "ต ซ." 
    text = text.replace(/หมู่ที่\s*([0-9]+)/g, 'หมู่ $1'); // เปลี่ยน หมู่ที่ X เป็น หมู่ X

    // 4. ***ทีเด็ด*** ลบข้อความอำเภอ/จังหวัดที่เบิ้ลซ้ำกันตอนท้าย (แก้ปัญหา Data จริง)
    // ตัวอย่าง: "อ.พระประแดง จ.สมุทรปราการ อ.พระประแดง จ.สมุทรปราการ"
    const duplicateTailRegex = /(อำเภอ[ก-ฮ]+(\s*จังหวัด[ก-ฮ]+)?)\s+\1/g;
    text = text.replace(duplicateTailRegex, '$1');

    // 5. ลบสัญลักษณ์ขยะและจัดช่องว่าง
    text = text.replace(/[,;|]/g, ' ');
    text = text.replace(/\s+/g, ' ').trim();

    return text;
  }

  function parseCoordinates(rawLatLong) {
    if (!rawLatLong) return null;
    const text = String(rawLatLong).trim();
    
    // รองรับวงเล็บ และรูปแบบเว้นวรรค
    const cleanText = text.replace(/[\(\)]/g, ''); 
    const coordMatch = cleanText.match(/([0-9]+\.[0-9]+)[,\s\|]+([0-9]+\.[0-9]+)/);
    
    if (coordMatch) {
      let lat = parseFloat(coordMatch[1]);
      let lng = parseFloat(coordMatch[2]);
      
      if (lat > 90 && lng < 30) {
        let temp = lat;
        lat = lng;
        lng = temp;
      }

      if (lat >= 5.6 && lat <= 20.5 && lng >= 97.3 && lng <= 105.7) {
        return { 
          lat: Number(lat.toFixed(6)), 
          lng: Number(lng.toFixed(6)) 
        };
      }
    }
    return null; 
  }

  return {
    normalizePersonName: normalizePersonName,
    normalizeAddress: normalizeAddress,
    parseCoordinates: parseCoordinates
  };

})();

lookupDestination(...): สมองกลตอนดึงข้อมูลไปใช้ (เอาชื่อ+ที่อยู่ ไปดึงพิกัดกลับมา)

/**
 * Module 10: Match Engine (สมองกลจับคู่และดึงข้อมูล)
 * หน้าที่: 
 * 1. นำข้อมูลดิบมาทำความสะอาดและสร้าง Master Data (Person, Place, Geo, Destination)
 * 2. รับข้อมูลชื่อ+ที่อยู่ (ที่ไม่มีพิกัด) มาค้นหาพิกัด Lat/Long กลับไปใช้งาน
 */

const MatchEngine = (function() {

  /**
   * สมองกลส่วนที่ 1: ใช้สร้าง/อัปเดต Master Data จากชีต SCG ที่มี Lat/Long จริง
   * (แก้ปัญหา 8 ข้อ ในขั้นตอนการจัดเก็บข้อมูล)
   */
  function buildMasterData(rawPersonName, rawAddress, rawLatLong, rawOwnerName) {
    // 1. ทำความสะอาดข้อมูลด่านแรก (Data Cleansing)
    const personInfo = NormalizeService.normalizePersonName(rawPersonName);
    const cleanAddress = NormalizeService.normalizeAddress(rawAddress);
    const coords = NormalizeService.parseCoordinates(rawLatLong);

    if (!coords) {
      // ถ้าไม่มีพิกัดจริง ถือว่าข้อมูลไม่สมบูรณ์พอที่จะสร้าง Master Data ได้
      return { success: false, reason: 'NO_LATLONG' };
    }

    // 2. จัดการ Geo (พิกัด) -> แก้ปัญหาข้อ 3, 8 (พิกัดซ้ำกันให้ใช้ ID เดียวกัน)
    // ค้นหาพิกัดในรัศมี 50 เมตร ถ้าเจอให้ใช้ Geo ID เดิม ถ้าไม่เจอสร้างใหม่
    const geoRecord = GeoService.findOrInsertGeoPoint(coords.lat, coords.lng);

    // 3. จัดการ Person (บุคคล) -> แก้ปัญหาข้อ 1, 4 (ชื่อเหมือนกัน/เขียนต่างกัน)
    // หาจากชื่อที่ Clean แล้ว หรือ Alias ถ้าไม่เจอสร้างใหม่
    const personRecord = PersonService.findOrInsertPerson(personInfo.cleanName, personInfo.extractedPhone);

    // 4. จัดการ Place (สถานที่) -> แก้ปัญหาข้อ 2, 5 (ที่อยู่เขียนต่างกันแต่ที่เดียวกัน)
    const placeRecord = PlaceService.findOrInsertPlace(cleanAddress, geoRecord.id);

    // 5. จัดการ Destination (ผูกความสัมพันธ์) -> แก้ปัญหาข้อ 6, 7 (มีหลายสาขา / 1 สถานที่มีหลายคน)
    // ผูก คน + สถานที่ + พิกัด เข้าด้วยกัน
    const destinationRecord = DestinationService.linkDestination(personRecord.id, placeRecord.id, geoRecord.id);

    return {
      success: true,
      destinationId: destinationRecord.id,
      geoId: geoRecord.id,
      lat: geoRecord.lat,
      lng: geoRecord.lng
    };
  }

  /**
   * สมองกลส่วนที่ 2: พระเอกของงาน (ใช้สำหรับ "ตารางงานประจำวัน")
   * รับข้อมูลที่สกปรก (ไม่มีพิกัด) แล้วดึง Lat/Long ที่ถูกต้องกลับไปให้
   */
  function lookupLatLongForDailyTask(rawPersonName, rawAddress) {
    // 1. ทำความสะอาดข้อความแบบเดียวกันมาตรฐานเดียวกับตอนสร้างฐาน
    const personInfo = NormalizeService.normalizePersonName(rawPersonName);
    const cleanAddress = NormalizeService.normalizeAddress(rawAddress);

    // 2. พยายามค้นหาบุคคล (จากชื่อตรงๆ หรือ Alias)
    const personId = PersonService.searchPerson(personInfo.cleanName);
    
    // 3. พยายามค้นหาสถานที่ (จากที่อยู่ตรงๆ หรือ Alias)
    const placeId = PlaceService.searchPlace(cleanAddress);

    // 4. ประมวลผลการดึง Lat/Long
    if (personId && placeId) {
      // กรณีที่ดีที่สุด: เจอทั้งคน และ สถานที่ -> ดึง Destination และ พิกัดเป๊ะๆ ได้เลย
      // (รองรับกรณีลูกค้า 1 คน มีหลายสาขา ระบบจะรู้ได้จาก PlaceId ว่าจะเอาพิกัดสาขาไหน)
      const destination = DestinationService.findByPersonAndPlace(personId, placeId);
      if (destination) {
        const geo = GeoService.getGeoPointById(destination.geoId);
        return {
          status: 'EXACT_MATCH',
          lat: geo.lat,
          lng: geo.lng,
          matchType: 'PERSON_AND_PLACE'
        };
      }
    }

    if (placeId) {
      // กรณีสำรองที่ 1: ชื่อคนอาจจะใหม่หรือไม่เคยมีในระบบ แต่ "สถานที่" (Place) นี้เราเคยไปส่งแล้วและมีพิกัด!
      // (แก้ปัญหาข้อ 5: คนละชื่อ แต่ที่อยู่เดียวกัน)
      const geoIds = DestinationService.getGeoIdsByPlace(placeId);
      if (geoIds.length > 0) {
        const geo = GeoService.getGeoPointById(geoIds[0]); // ดึงพิกัดของสถานที่นี้
        return {
          status: 'PLACE_MATCH', // แจ้งเตือนว่าแมตช์แค่สถานที่นะ (ชื่อคนใหม่อาจจะเป็นลูกน้องเขารับแทน)
          lat: geo.lat,
          lng: geo.lng,
          matchType: 'PLACE_ONLY'
        };
      }
    }

    if (personId) {
      // กรณีสำรองที่ 2: ไม่รู้จักสถานที่ แต่คุ้นชื่อลูกค้ารายนี้ (อาจจะมีการเปลี่ยนการพิมพ์ที่อยู่)
      const geoIds = DestinationService.getGeoIdsByPerson(personId);
      if (geoIds.length === 1) {
        // ถ้าลูกค้ารายนี้เคยส่งแค่ที่เดียวตลอด ให้สันนิษฐานว่าเป็นพิกัดนั้น
        const geo = GeoService.getGeoPointById(geoIds[0]);
        return {
          status: 'PERSON_MATCH',
          lat: geo.lat,
          lng: geo.lng,
          matchType: 'PERSON_ONLY'
        };
      } else if (geoIds.length > 1) {
        // ถ้าลูกค้าคนนี้มีหลายสาขา แต่พิมพ์ที่อยู่ใหม่มามั่วมากจนระบบเดาสาขาไม่ออก
        return {
          status: 'MULTIPLE_BRANCHES_FOUND',
          lat: null,
          lng: null,
          message: 'ลูกค้ามีหลายสาขา กรุณาตรวจสอบที่อยู่'
        };
      }
    }

    // กรณีหาไม่เจอเลย (ลูกค้าใหม่เอี่ยม) -> ต้องให้ Admin รีวิวหรือโทรถามพิกัด
    return {
      status: 'NOT_FOUND',
      lat: null,
      lng: null
    };
  }

  // --- Public API ---
  return {
    buildMasterData: buildMasterData,
    lookupLatLongForDailyTask: lookupLatLongForDailyTask
  };

Person Service
27 เม.ย. 17:03
/**
 * Module 06: Person Service (Identity & Alias Management)
 * หน้าที่: ค้นหา, ตรวจสอบเบอร์โทร, สร้าง ID บุคคล (PER-...) และจัดการชื่อพ้อง (Alias)
 */

const PersonService = (function() {
  
  // โหลดข้อมูลเข้า Memory เพื่อให้ทำงานเร็วขึ้น (ไม่ต้องดึง Sheet ทุกบรรทัด)
  // ในระบบจริง จะถูกเรียกตอนเริ่มต้นโปรแกรม
  let personMasterData = []; // เก็บข้อมูลจากชีต M_PERSON
  let personAliasData = [];  // เก็บข้อมูลจากชีต M_PERSON_ALIAS

  /**
   * จำลองฟังก์ชันโหลดข้อมูล (คุณนำไปปรับใช้กับ Utils ที่ดึงข้อมูลจากชีตได้)
   */
  function loadData(personData, aliasData) {
    personMasterData = personData;
    personAliasData = aliasData;
  }

  /**
   * 🎯 ฟังก์ชันหลัก: หา ID ลูกค้า หรือ สร้างใหม่ถ้าไม่เคยเจอ
   * @param {Object} personObj - Object ที่ผ่านการทำความสะอาดจาก 05_NormalizeService แล้ว
   * @returns {Object} { id: 'PER-XXXX', isNew: boolean }
   */
  function findOrInsertPerson(personObj) {
    if (!personObj || !personObj.cleanName) {
      return null;
    }

    const cleanName = personObj.cleanName;
    const phone = personObj.extractedPhone;
    const rawName = personObj.originalName;

    // --- สเตป 1: ค้นหาจากเบอร์โทรศัพท์ (ไม้ตายสำคัญ!) ---
    // ถ้ามีเบอร์โทร ให้เอาเบอร์โทรไปค้นหาก่อน เพราะเบอร์โทรระบุตัวตนได้แม่นกว่าชื่อที่พิมพ์ผิดๆ ถูกๆ
    if (phone) {
      const matchByPhone = personMasterData.find(p => p.phone === phone);
      if (matchByPhone) {
        // เจอคนจากเบอร์โทร! ให้เรียนรู้ชื่อที่พิมพ์มาใหม่นี้เป็น Alias ซะ
        learnNewAlias(matchByPhone.person_id, rawName, cleanName);
        return { id: matchByPhone.person_id, isNew: false, matchType: 'PHONE' };
      }
    }

    // --- สเตป 2: ค้นหาจากชื่อที่สะอาดแล้ว (Clean Name) ในตาราง Master ---
    const matchByCleanName = personMasterData.find(p => p.person_name_normalized === cleanName);
    if (matchByCleanName) {
      // อัปเดตเบอร์โทรเข้าระบบ (ถ้า master ยังไม่มีเบอร์ แต่บิลนี้มีเบอร์มาให้)
      updatePhoneIfMissing(matchByCleanName, phone);
      learnNewAlias(matchByCleanName.person_id, rawName, cleanName);

    }

    // --- สเตป 3: ค้นหาจากชื่อพ้อง (Alias) ---
    // เผื่อลูกค้าเคยพิมพ์ชื่อแปลกๆ แล้วระบบเคยจำไว้
    const matchByAlias = personAliasData.find(a => a.alias_normalized === cleanName || a.alias_raw === rawName);

    }

      person_name_normalized: cleanName,
      phone: phone || '',
      is_company: personObj.isCompany ? 'Y' : 'N',
      first_seen_date: new Date().toISOString(),
      usage_count: 1
    };
    
    // บันทึกลง Memory (และคุณต้องเขียนโค้ดบันทึกลง Sheet ต่อไป)
    personMasterData.push(newPersonRecord);
    savePersonToSheet(newPersonRecord); // ฟังก์ชันสมมติสำหรับเขียนลงชีต

    learnNewAlias(newPersonId, rawName, cleanName);

    return { id: newPersonId, isNew: true, matchType: 'NEW' };
  }

  /**

/**
 * Module 05: Normalize Service (Data Cleansing Engine)
 * หน้าที่: ทำความสะอาด สกัด และจัดรูปแบบข้อมูลดิบที่มั่วและไม่เป็นระเบียบ
 * ให้กลายเป็นข้อมูลมาตรฐานก่อนส่งเข้าสู่ Match Engine
 */

const NormalizeService = (function() {
  
  // --- Dictionaries สำหรับการทำความสะอาดข้อมูล ---
  
  // คำนำหน้าชื่อบุคคลที่ต้องตัดทิ้ง เพื่อหาแก่นของชื่อจริงๆ
  const PERSON_PREFIXES = [
    'คุณ', 'นาย', 'นางสาว', 'น.ส.', 'นาง', 
    'พี่', 'น้อง', 'เฮีย', 'เจ๊', 'ซ้อ', 'เตี่ย',
    'ช่าง', 'ลุง', 'ป้า', 'น้า', 'อา', 'ตา', 'ยาย',
    'หมอ', 'นพ.', 'พญ.', 'ดร.', 'อ.'
  ];

  // คำที่บ่งบอกว่านี่คือ "นิติบุคคล" หรือ "สถานที่" ไม่ใช่ชื่อคน
  const COMPANY_KEYWORDS = [
    'บจก.', 'บจก', 'บริษัท', 'หจก.', 'หจก', 'ห้างหุ้นส่วน', 
    'บมจ.', 'ร้าน', 'คลินิก', 'รพ.', 'โรงพยาบาล', 
    'บ.', 'จำกัด', 'มหาชน', 'คลังสินค้า', 'โกดัง', 'สน.'
  ];

  // คำย่อในที่อยู่ที่ต้องขยายให้เป็นคำเต็ม
  const ADDRESS_REPLACEMENTS = {
    'ถ.': 'ถนน ',
    'ซ.': 'ซอย ',
    'ม.': 'หมู่ ',
    'หมู่ที่ ': 'หมู่ ',
    'ต.': 'ตำบล ',
    'อ.': 'อำเภอ ',
    'จ.': 'จังหวัด ',
    'กทม.': 'กรุงเทพมหานคร',
    'กทม': 'กรุงเทพมหานคร'
  };

  /**
   * ล้างข้อมูลและดึงข้อมูลที่ซ่อนอยู่ออกจากคอลัมน์ "ชื่อปลายทาง"
   * @param {string} rawName - ข้อมูลดิบจากคอลัมน์ "ชื่อปลายทาง"
   * @returns {Object} - คืนค่าเป็น Object ที่แยกส่วนข้อมูลแล้ว
   */
  function normalizePersonName(rawName) {
    if (!rawName || typeof rawName !== 'string') {
      return { cleanName: '', isCompany: false, extractedPhone: null, originalName: rawName || '' };
    }

    let text = rawName.trim();
    let isCompany = false;
    let extractedPhone = null;

    // 1. ตรวจจับและดึงเบอร์โทรศัพท์ที่ปนมากับชื่อ (รูปแบบเบอร์มือถือ/บ้าน 9-10 หลัก)
    // ลบขีด หรือช่องว่างในเบอร์ก่อน
    const phoneRegex = /(?:โทร\.?|Tel\.?|เบอร์)?\s*([0-9]{2,3}[-\s]?[0-9]{3}[-\s]?[0-9]{4})/i;
    const phoneMatch = text.match(phoneRegex);
    if (phoneMatch) {
      extractedPhone = phoneMatch[1].replace(/[-\s]/g, ''); // เก็บเฉพาะตัวเลข
      text = text.replace(phoneMatch[0], ''); // เอาเบอร์โทรออกจากชื่อ
    }

    // 2. เช็คว่าเป็นนิติบุคคลหรือไม่
    const upperText = text.toUpperCase();
    for (const keyword of COMPANY_KEYWORDS) {
      if (upperText.includes(keyword)) {
        isCompany = true;
        break; // ถ้าเจอว่าเป็นบริษัทแล้ว ออกจากลูปได้เลย
      }
    }

    // 3. ทำความสะอาดคำนำหน้าชื่อบุคคล (ทำเฉพาะกรณีไม่ใช่บริษัท เพื่อป้องกันการตัดคำผิดพลาด)
    if (!isCompany) {
      // สร้าง Regex แบบ Dynamic จาก Array (เช่น ^(คุณ|นาย|ช่าง)\s*)
      const prefixPattern = new RegExp(`^(${PERSON_PREFIXES.join('|')})\\s*`, 'i');
      text = text.replace(prefixPattern, '');
    }

    // 4. ลบคำต่อท้ายที่ไม่มีความหมาย หรือคำที่เกิดจากการตัดคำมามั่วๆ
    const suffixesToRemove = ['รับของ', 'ฝากส่ง', 'โทร.', 'Tel.'];
    suffixesToRemove.forEach(suffix => {
      const suffixRegex = new RegExp(`\\s*${suffix}$`, 'i');
      text = text.replace(suffixRegex, '');
    });

    // 5. ลบช่องว่างที่ซ้ำซ้อนกันเกิน 1 เคาะ (Double spaces) และอักขระพิเศษบางตัว
    text = text.replace(/[\(\)\[\]]/g, ' '); // เปลี่ยนวงเล็บเป็นช่องว่าง
    text = text.replace(/\s+/g, ' ').trim();

    return {
      cleanName: text,
      isCompany: isCompany,
      extractedPhone: extractedPhone,
      originalName: rawName.trim()
    };
  }

  /**
   * แปลงข้อมูลที่อยู่ให้เป็นมาตรฐาน (Standardization)
   * @param {string} rawAddress - ข้อมูลดิบจากคอลัมน์ "ที่อยู่ปลายทาง"
   * @returns {string} - ที่อยู่ที่จัดรูปแบบแล้ว
   */
  function normalizeAddress(rawAddress) {
    if (!rawAddress || typeof rawAddress !== 'string') return '';

    let text = rawAddress.trim();

    // 1. แปลงคำย่อเป็นคำเต็ม (ถ., ซ., จ.)
    for (const [abbr, full] of Object.entries(ADDRESS_REPLACEMENTS)) {
      // ใช้ Regex ห้ามแทนที่ถ้าเป็นส่วนหนึ่งของคำอื่น (ต้องเว้นวรรค หรืออยู่หน้าข้อความ)
      // อนุโลมให้ไม่ต้องมี space ตามหลังจุดก็ได้ เพราะคนชอบพิมพ์ติดกัน
      const regex = new RegExp(abbr.replace('.', '\\.'), 'g');
      text = text.replace(regex, full);
    }

    // 2. ลบคำซ้ำซ้อนที่มักเจอในระบบเก่า (เช่น อ.เมืองสมุทรสาคร จ.สมุทรสาคร -> อำเภอเมือง จังหวัดสมุทรสาคร)
    text = text.replace(/อำเภอเมือง([ก-ฮ]+)/g, 'อำเภอเมือง จังหวัด$1');
    text = text.replace(/เขตเขต/g, 'เขต'); // แก้เคสพิมพ์ซ้ำ "เขตเขตหนองแขม"
    text = text.replace(/ตำบลตำบล/g, 'ตำบล');

    // 3. ลบคำว่า "ประเทศไทย" หรือ "Thailand" ออกไปเลย เพราะทำให้ Match ยากขึ้นและไม่จำเป็น
    text = text.replace(/\s*ประเทศไทย\s*/g, ' ');
    text = text.replace(/\s*Thailand\s*/gi, ' ');

    // 4. ทำความสะอาดช่องว่างและสัญลักษณ์ขยะ
    text = text.replace(/[,;]/g, ' '); // เอาลูกน้ำออก
    text = text.replace(/\s+/g, ' ').trim(); // ลดช่องว่างให้เหลือเคาะเดียว

    return text;
  }

  /**
   * จัดการกับข้อมูลจุดพิกัด (Lat/Long) ที่อาจมาในรูปแบบข้อความมั่วๆ
   * @param {string|number} rawLatLong - ข้อความพิกัด เช่น "13.123, 100.123"
   * @returns {Object|null} - คืนค่า {lat: number, lng: number} หรือ null ถ้าข้อมูลเสีย
   */
  function parseCoordinates(rawLatLong) {
    if (!rawLatLong) return null;
    
    const text = String(rawLatLong).trim();
    // หาตัวเลขทศนิยม 2 ชุดที่คั่นด้วยลูกน้ำหรือช่องว่าง
    const coordMatch = text.match(/([0-9]+\.[0-9]+)[,\s]+([0-9]+\.[0-9]+)/);
    
    if (coordMatch) {
      const lat = parseFloat(coordMatch[1]);
      const lng = parseFloat(coordMatch[2]);
      
      // Basic Sanity Check ว่าพิกัดอยู่ในประเทศไทยคร่าวๆ (Lat 5-21, Lng 97-106)
      if (lat > 5 && lat < 21 && lng > 97 && lng < 106) {
        return { lat: lat, lng: lng };
      }
    }
    return null;
  }

  // --- Public API ---
  return {
    normalizePersonName: normalizePersonName,
    normalizeAddress: normalizeAddress,
    parseCoordinates: parseCoordinates
  };

})();

      }
    }

    if (!isCompany) {

    text = text.replace(/เขตเขต/g, 'เขต');

    text = text.replace(/จังหวัดจังหวัด/g, 'จังหวัด');

    // 3. ถอดคำว่า "ประเทศไทย" หรือ "Thailand" ออก (รบกวน MatchEngine)
    text = text.replace(/\s*ประเทศไทย\s*/g, ' ');
    text = text.replace(/\s*Thailand\s*/gi, ' ');

    // 4. ซ่อมแซมคำว่า "หมู่ที่ X" ให้เหลือแค่ "หมู่ X" เพื่อความเป็นเอกภาพ
    text = text.replace(/หมู่ที่\s*([0-9]+)/g, 'หมู่ $1');

    // 5. ลบสัญลักษณ์ขยะและจัดช่องว่าง
    text = text.replace(/[,;]/g, ' ');
    text = text.replace(/\s+/g, ' ').trim();

    return text;
  }

   * @param {string|number} rawLatLong - พิกัดดิบ (เช่น "13.123, 100.123")

      let lng = parseFloat(coordMatch[2]);

      if (lat >= 5.6 && lat <= 20.5 && lng >= 97.3 && lng <= 105.7) {

          lat: Number(lat.toFixed(6)), 
          lng: Number(lng.toFixed(6)) 
        };
      }
    }
    return null; // ข้อมูลพิกัดเสีย หรือไม่อยู่ในไทย
  }

  // ==========================================
  // 🚪 EXPORT PUBLIC API
  // ==========================================
  return {

    normalizeAddress: normalizeAddress,
    parseCoordinates: parseCoordinates
  };

    'ธนาคาร', 'สาขา', 'ศูนย์', 'ฟาร์ม', 'อู่', 'ตลาด', 'มูลนิธิ', 'สมาคม', 
    'กรม', 'กระทรวง', 'สหกรณ์', 'การเคหะ', 'ที่ว่าการ', 'ศาลากลาง',
    'คอนโด', 'นิติบุคคล', 'หมู่บ้าน', 'มบ.', 'รีสอร์ท', 'โรงแรม', 'แมนชั่น', 'อพาร์ทเม้นท์', 'หอพัก', 'พลาซ่า', 'เซ็นเตอร์'

  const DELIVERY_NOTES_KEYWORDS = [
    'เก็บเงินปลายทาง', 'cod', 'โอนแล้ว', 'จ่ายแล้ว', 'ไม่ต้องโทร', 'โทรแจ้งก่อน', 'โทรด่วน',

    { regex: /ถ\./g, replace: 'ถนน ' },

  function normalizePersonName(rawName) {
    if (!rawName || typeof rawName !== 'string') {
      return { cleanName: '', isCompany: false, extractedPhone: null, deliveryNotes: [], extractedDocNo: null, originalName: rawName || '' };
    }

    let text = rawName.trim();
    let isCompany = false;
    let extractedPhone = null;
    let extractedDocNo = null;
    let deliveryNotes = [];

    // --- STEP 1: สกัดรหัสเอกสาร (DN, PO) ที่ปนมากับชื่อ (เจอใน Data จริง เช่น "-DN.2250966887") ---
    const docRegex = /[-\s]*(?:DN|PO|SO|INV|บิล)\.?\s*([A-Z0-9]+)/i;
    const docMatch = text.match(docRegex);
    if (docMatch) {
      extractedDocNo = docMatch[0].replace(/[-\s]/g, ''); // เก็บไว้เผื่อค้นหา
      text = text.replace(docMatch[0], ''); 
    }

    // --- STEP 2: สกัดเบอร์โทรศัพท์ออก (รองรับรูปแบบติดกัน หรือเว้นวรรค) ---
    const phoneRegex = /(?:โทร\.?|Tel\.?|เบอร์)?\s*(0[0-9]{1,2}[-\s]?[0-9]{3}[-\s]?[0-9]{4})/i;
    const phoneMatch = text.match(phoneRegex);
    if (phoneMatch) {
      extractedPhone = phoneMatch[1].replace(/[-\s]/g, ''); 
      text = text.replace(phoneMatch[0], ''); 
    }

    // --- STEP 3: สกัด "คำสั่งจัดส่ง" ออกมาเก็บไว้เป็น Meta-data ---
    DELIVERY_NOTES_KEYWORDS.forEach(note => {
      const noteRegex = new RegExp(note, 'gi');
      if (noteRegex.test(text)) {
        deliveryNotes.push(note);
        text = text.replace(noteRegex, ''); 
      }
    });

    // --- STEP 4: ตรวจสอบสถานะ "นิติบุคคล" หรือ "สถานที่" ---
    const upperText = text.toUpperCase();
    for (const keyword of COMPANY_KEYWORDS) {
      if (upperText.includes(keyword)) {
        isCompany = true;
        break; 
      }
    }

    // --- STEP 5: ตัดคำนำหน้าบุคคล (เฉพาะกรณีที่ไม่ใช่บริษัท) ---
    if (!isCompany) {
      const originalBeforePrefixRemoval = text;
      const escapedPrefixes = SORTED_PREFIXES.map(p => p.replace(/\./g, '\\.'));
      const prefixPattern = new RegExp(`^(${escapedPrefixes.join('|')})\\s*`, 'i');
      text = text.replace(prefixPattern, '');

      // ป้องกันกรณีชื่อหายหมด (เช่น พิมพ์มาแค่ "เฮีย")
      if (text.trim() === '') {
        text = originalBeforePrefixRemoval;
      }
    }

    // --- STEP 6: เก็บกวาดขยะขั้นสุดท้าย ---
    const junkSuffixes = ['รับของ', 'ฝากส่ง', 'ติดต่อ', 'ผู้รับ', 'ลูกค้า', 'ส่ง', 'โทรเข้า', 'โทร'];
    junkSuffixes.forEach(junk => {
      const junkRegex = new RegExp(`\\s*${junk}$`, 'i');
      text = text.replace(junkRegex, '');
    });

    text = text.replace(/[\(\)\[\]\-]/g, ' '); 
    text = text.replace(/\s+/g, ' ').trim();

    return {
      cleanName: text,
      isCompany: isCompany,
      extractedPhone: extractedPhone,
      extractedDocNo: extractedDocNo,
      deliveryNotes: deliveryNotes.join(', '), 
      originalName: rawName.trim()
    };
  }

  function normalizeAddress(rawAddress) {
    if (!rawAddress || typeof rawAddress !== 'string') return '';

    let text = rawAddress.trim();

    // 1. แปลงคำย่อเป็นคำเต็ม
    ADDRESS_REPLACEMENTS.forEach(replacer => {
      text = text.replace(replacer.regex, replacer.replace);
    });

    // 2. ลบคำว่าประเทศไทยและรหัสไปรษณีย์ที่อยู่ท้ายสุด (มักมาจาก ชื่อที่อยู่จาก_LatLong)
    text = text.replace(/\s*ประเทศไทย\s*/g, ' ');
    text = text.replace(/\s*Thailand\s*/gi, ' ');
    
    // 3. ซ่อมแซมคำพิมพ์ผิดซ้ำซ้อน (เจอใน Data จริง)
    text = text.replace(/เขตเขต/g, 'เขต');
    text = text.replace(/ตำบลตำบล/g, 'ตำบล');
    text = text.replace(/อำเภออำเภอ/g, 'อำเภอ');
    text = text.replace(/จังหวัดจังหวัด/g, 'จังหวัด');
    text = text.replace(/ต ซ\./g, 'ซอย'); // แก้ไขคนพิมพ์มั่ว "ต ซ." 
    text = text.replace(/หมู่ที่\s*([0-9]+)/g, 'หมู่ $1'); // เปลี่ยน หมู่ที่ X เป็น หมู่ X

    // 4. ***ทีเด็ด*** ลบข้อความอำเภอ/จังหวัดที่เบิ้ลซ้ำกันตอนท้าย (แก้ปัญหา Data จริง)
    // ตัวอย่าง: "อ.พระประแดง จ.สมุทรปราการ อ.พระประแดง จ.สมุทรปราการ"
    const duplicateTailRegex = /(อำเภอ[ก-ฮ]+(\s*จังหวัด[ก-ฮ]+)?)\s+\1/g;
    text = text.replace(duplicateTailRegex, '$1');

    // 5. ลบสัญลักษณ์ขยะและจัดช่องว่าง
    text = text.replace(/[,;|]/g, ' ');
    text = text.replace(/\s+/g, ' ').trim();

    return text;
  }

  function parseCoordinates(rawLatLong) {
    if (!rawLatLong) return null;
    const text = String(rawLatLong).trim();
    
    // รองรับวงเล็บ และรูปแบบเว้นวรรค
    const cleanText = text.replace(/[\(\)]/g, ''); 
    const coordMatch = cleanText.match(/([0-9]+\.[0-9]+)[,\s\|]+([0-9]+\.[0-9]+)/);
    
    if (coordMatch) {
      let lat = parseFloat(coordMatch[1]);
      let lng = parseFloat(coordMatch[2]);
      
      if (lat > 90 && lng < 30) {
        let temp = lat;
        lat = lng;
        lng = temp;
      }

      if (lat >= 5.6 && lat <= 20.5 && lng >= 97.3 && lng <= 105.7) {
        return { 
          lat: Number(lat.toFixed(6)), 
          lng: Number(lng.toFixed(6)) 
        };
      }
    }
    return null; 
  }

  return {
    normalizePersonName: normalizePersonName,
    normalizeAddress: normalizeAddress,
    parseCoordinates: parseCoordinates
  };

})();



/**
 * Module 08: Geo Service (Spatial & Coordinate Management)
 * หน้าที่: จัดการพิกัด Lat/Long, คำนวณระยะทางบนผิวโลก (Haversine), และป้องกันพิกัดซ้ำซ้อนในรัศมี
 */

const GeoService = (function() {
  
  let geoMasterData = []; // เก็บข้อมูลจากชีต M_GEO_POINT

  // ค่าความคลาดเคลื่อนที่ยอมรับได้ (เมตร) - สมมติคนขับเดินจากรถไปหน้าบ้าน
  // ดึงจาก Config ได้ แต่ hardcode ไว้เป็น Fallback
  const ACCEPTABLE_RADIUS_METERS = 50; 

  function loadData(geoData) {
    geoMasterData = geoData;
  }

  /**
   * 🎯 หา Geo ID เดิมถ้อยู่ในรัศมี หรือสร้างพิกัดใหม่
   * @param {number} lat - ละติจูด (ทศนิยม 6 ตำแหน่ง)
   * @param {number} lng - ลองจิจูด (ทศนิยม 6 ตำแหน่ง)
   * @param {string} addressFromLatLong - "ชื่อที่อยู่จาก_LatLong" (ถ้ามี)
   * @returns {Object} { id: 'GEO-XXXX', isNew: boolean, lat: number, lng: number }
   */
  function findOrInsertGeoPoint(lat, lng, addressFromLatLong = '') {
    if (!lat || !lng) return null;

    // --- สเตป 1: Fast Grid Search (คัดกรองเบื้องต้นเพื่อความเร็ว) ---
    // สร้าง Grid Key หยาบๆ (ทศนิยม 3 ตำแหน่ง = พื้นที่ประมาณ 111x111 เมตร)
    // เพื่อไม่ต้องเอาพิกัดนี้ไปคำนวณ Haversine กับพิกัดทั้งประเทศ (O(n) -> O(1))
    const gridKey = lat.toFixed(3) + '_' + lng.toFixed(3);
    
    // หาพิกัดใน Master ที่อยู่ใน Grid เดียวกัน หรือ Grid ติดกัน
    const candidates = geoMasterData.filter(g => 
      g.geo_key_3 === gridKey || isNeighborGrid(lat, lng, g.lat_norm, g.long_norm)
    );

    // --- สเตป 2: Precise Calculation (คำนวณระยะทางจริงบนผิวโลก) ---
    for (const candidate of candidates) {
      const distance = calculateHaversineDistance(lat, lng, candidate.lat_norm, candidate.long_norm);
      
      // ถ้าระยะทางน้อยกว่า 50 เมตร ถือว่าเป็น "จุดส่งเดียวกัน (Geo ID เดียวกัน)" (แก้ปัญหาข้อ 3, 8)
      if (distance <= ACCEPTABLE_RADIUS_METERS) {
        
        // อัปเดตที่อยู่จาก LatLong ถ้าระบบเพิ่งหามาได้ (เติมเต็มข้อมูล)
        if (addressFromLatLong && !candidate.address_from_latlong_best) {
           candidate.address_from_latlong_best = addressFromLatLong;
           // TODO: อัปเดตกลับลง Sheet
        }

        return { id: candidate.geo_id, isNew: false, lat: candidate.lat_norm, lng: candidate.long_norm, distance: distance };
      }
    }

    // --- สเตป 3: ถ้าไม่อยู่ในรัศมีใดๆ เลย ถือเป็นพิกัดใหม่บนโลก ---
    const newGeoId = generateGeoId();
    const newGeoRecord = {
      geo_id: newGeoId,
      lat_raw: lat,
      long_raw: lng,
      lat_norm: lat,
      long_norm: lng,
      geo_key_3: gridKey, // เก็บ Grid ไว้ค้นหาเร็วๆ
      address_from_latlong_best: addressFromLatLong,
      first_seen_date: new Date().toISOString(),
      usage_count: 1
    };

    geoMasterData.push(newGeoRecord);
    saveGeoToSheet(newGeoRecord);

    return { id: newGeoId, isNew: true, lat: lat, lng: lng, distance: 0 };
  }

  /**
   * 🧮 สูตรคำนวณระยะทางบนผิวโลก (Haversine Formula) คืนค่าเป็น "เมตร"
   * แม่นยำระดับวิศวกรรมโยธา
   */
  function calculateHaversineDistance(lat1, lon1, lat2, lon2) {
    const R = 6371e3; // รัศมีโลก (เมตร)
    const φ1 = lat1 * Math.PI/180;
    const φ2 = lat2 * Math.PI/180;
    const Δφ = (lat2-lat1) * Math.PI/180;
    const Δλ = (lon2-lon1) * Math.PI/180;

    const a = Math.sin(Δφ/2) * Math.sin(Δφ/2) +
              Math.cos(φ1) * Math.cos(φ2) *
              Math.sin(Δλ/2) * Math.sin(Δλ/2);
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));

    return R * c; // ระยะทางเป็นเมตร
  }

  // ตัวช่วยคำนวณ Grid ใกล้เคียง (ลดภาระ CPU)
  function isNeighborGrid(lat1, lng1, lat2, lng2) {
    return Math.abs(lat1 - lat2) <= 0.002 && Math.abs(lng1 - lng2) <= 0.002;
  }

  function generateGeoId() {
    return 'GEO-' + Utilities.getUuid().substring(0, 8).toUpperCase();
  }

  function getGeoPointById(geoId) {
    return geoMasterData.find(g => g.geo_id === geoId);
  }

  function saveGeoToSheet(record) { /* บันทึกลง Sheet M_GEO_POINT */ }

  return {
    loadData: loadData,
    findOrInsertGeoPoint: findOrInsertGeoPoint,
    calculateDistance: calculateHaversineDistance,
    getGeoPointById: getGeoPointById
  };

})();

  /**
   * 🎯 หาสถานที่เดิม หรือ สร้างใหม่ (เชื่อมโยงความฉลาดจาก Geo ID)
   * @param {string} cleanAddress - ที่อยู่ที่ซักล้างแล้วจาก 05_Normalize

  function findOrInsertPlace(cleanAddress, geoId, rawAddress) {

    const exactMatch = placeMasterData.find(p => p.place_name_normalized === cleanAddress);
    if (exactMatch) {

      return { id: exactMatch.place_id, isNew: false, matchType: 'EXACT' };
    }

    // --- สเตป 2: หาจากประวัติความมั่ว (Alias Match) ---
    const aliasMatch = placeAliasData.find(a => a.alias_normalized === cleanAddress || a.alias_raw === rawAddress);
    if (aliasMatch) {
      return { id: aliasMatch.place_id, isNew: false, matchType: 'ALIAS' };
    }

    // --- สเตป 3: THE ULTIMATE TRICK (Fuzzy + Geo Binding) ---
    // ถ้าข้อความไม่ตรงเป๊ะ แต่ "พิกัด (Geo ID) ดันเป็นที่เดียวกัน!" 
    // เราจะเอาข้อความมาเทียบคะแนนความเหมือนกัน ถ้าระดับนึง ถือว่าเป็นที่เดียวกันแค่พิมพ์ผิด!
    
    // 3.1 หา Place ทั้งหมดที่เคยไปส่งที่ Geo ID นี้ (ต้องเรียกข้ามไปหา DestinationService สมมติว่ามีฟังก์ชันนี้)
    // *หมายเหตุ: ในโครงสร้างจริง DestinationService จะส่ง PlaceIds ที่ผูกกับ GeoId นี้มาให้*
    const candidatePlacesAtThisGeo = getPlaceIdsByGeoId_Mock(geoId); 
    
    if (candidatePlacesAtThisGeo.length > 0) {
      for (const pId of candidatePlacesAtThisGeo) {

        if (masterPlace) {
          // เทียบคะแนนความเหมือนของข้อความ (เช่น 110/2 ม.8 ต.บางหัวเสือ vs 110/2 หมู่ 8 ซ.สยามพัฒนา)
          const similarityScore = calculateTextSimilarity(cleanAddress, masterPlace.place_name_normalized);

          if (similarityScore > 0.65) {

            return { id: masterPlace.place_id, isNew: false, matchType: 'FUZZY_GEO_BIND' };
          }
        }
      }
    }

    const newPlaceId = generatePlaceId();

      place_name_canonical: cleanAddress,
      place_name_normalized: cleanAddress,
      address_best: cleanAddress,
      first_seen_date: new Date().toISOString(),
      usage_count: 1
    };

    placeMasterData.push(newPlaceRecord);
    savePlaceToSheet(newPlaceRecord);
    learnNewPlaceAlias(newPlaceId, rawAddress, cleanAddress);

    return { id: newPlaceId, isNew: true, matchType: 'NEW' };
  }

  /**
   * เรียนรู้ชื่อสถานที่แปลกๆ เข้าตาราง Alias
   */
  function learnNewPlaceAlias(placeId, rawAddress, cleanAddress) {
    const exists = placeAliasData.find(a => a.place_id === placeId && a.alias_raw === rawAddress);
    if (!exists && rawAddress && cleanAddress) {
      const newAlias = {
        place_alias_id: 'L_AL-' + Utilities.getUuid().substring(0,8).toUpperCase(),
        place_id: placeId,
        alias_raw: rawAddress,
        alias_normalized: cleanAddress,
        source_field: 'SYSTEM'
      };
      placeAliasData.push(newAlias);
      saveAliasToSheet(newAlias);
    }
  }

  /**
   * 🧠 NLP ขนาดย่อม: คำนวณความคล้ายคลึงของข้อความ (Sørensen–Dice coefficient)
   * เหมาะกับภาษาไทยที่ตัดคำได้บ้างไม่ได้บ้าง
   */
  function calculateTextSimilarity(str1, str2) {
    if (str1 === str2) return 1.0;
    if (!str1 || !str2) return 0.0;

    // แตกเป็นคู่ตัวอักษร (Bigrams)
    function getBigrams(str) {
      const bigrams = new Set();
      for (let i = 0; i < str.length - 1; i++) {
        bigrams.add(str.substring(i, i + 2));
      }
      return bigrams;
    }

    const set1 = getBigrams(str1);
    const set2 = getBigrams(str2);
    
    let intersection = 0;
    for (const bg of set1) {
      if (set2.has(bg)) intersection++;
    }

    // คืนค่าระหว่าง 0.0 (ไม่เหมือนเลย) ถึง 1.0 (เหมือนเป๊ะ)
    return (2.0 * intersection) / (set1.size + set2.size);
  }

/**
 * Module 09: Destination Service (The Grand Synergy)
 * หน้าที่: ผูกความสัมพันธ์ระหว่าง Person (บุคคล), Place (สถานที่), และ Geo (พิกัด)
 * ให้กลายเป็น 1 จุดส่งสินค้า (Destination) เพื่อใช้เป็นฐานข้อมูลดึง Lat/Long ในอนาคต
 */

const DestinationService = (function() {
  
  let destinationMasterData = []; // เก็บข้อมูลจากชีต M_DESTINATION

  function loadData(destinationData) {
    destinationMasterData = destinationData;
  }

  /**
   * 🎯 1. ฟังก์ชันสร้าง/อัปเดต จุดส่งสินค้า (ใช้ตอนสอนระบบด้วย Data จริง)
   * @param {string} personId - จาก 06_PersonService
   * @param {string} placeId - จาก 07_PlaceService
   * @param {string} geoId - จาก 08_GeoService
   * @param {string} personName - ชื่อคน (เพื่อสร้าง Label ให้อ่านง่าย)
   * @param {string} placeName - ชื่อสถานที่ (เพื่อสร้าง Label ให้อ่านง่าย)
   * @returns {Object} Destination Record
   */
  function linkDestination(personId, placeId, geoId, personName = '', placeName = '') {
    if (!personId || !placeId || !geoId) return null;

    // สร้าง Key เฉพาะตัว (Unique Key) เพื่อป้องกันการสร้างข้อมูลซ้ำซ้อน
    // กฎ: 1 คน + 1 สถานที่ + 1 พิกัด = 1 Destination เสมอ
    const destinationKey = `${personId}|${placeId}|${geoId}`;

    // ค้นหาว่าเคยมีการผูกความสัมพันธ์นี้ไว้แล้วหรือยัง
    const existingDest = destinationMasterData.find(d => d.destination_key === destinationKey);

    if (existingDest) {
      // ถ้ามีแล้ว: ไม่อัปเดตพิกัด แต่ "เพิ่มยอดการใช้งาน (Usage Count)" 
      // เพื่อให้ระบบรู้ว่าจุดนี้มีความน่าเชื่อถือสูงมาก ไปส่งบ่อย
      existingDest.usage_count = (existingDest.usage_count || 0) + 1;
      existingDest.last_seen_date = new Date().toISOString();
      
      // สมมติฟังก์ชันอัปเดตลง Sheet
      updateDestinationInSheet(existingDest);
      
      return { id: existingDest.destination_id, isNew: false, record: existingDest };
    }

    // ถ้ายังไม่เคยมี: สร้าง Destination ใหม่แกะกล่อง
    const newDestId = generateDestinationId();
    
    // สร้าง Label ให้อ่านง่ายสำหรับ Admin (เช่น "ฟิมล์ @ หมู่บ้าน VILLAGIO")
    const label = (personName && placeName) 
      ? `${personName} @ ${placeName.substring(0, 30)}...` 
      : `Destination ${newDestId}`;

    const newDestinationRecord = {
      destination_id: newDestId,
      person_id: personId,
      place_id: placeId,
      geo_id: geoId,
      destination_label_canonical: label,
      destination_key: destinationKey,
      confidence_status: 'HIGH', // ค่าเริ่มต้นคือสูง เพราะมาจากพิกัดที่คนขับกดส่งจริง
      first_seen_date: new Date().toISOString(),
      last_seen_date: new Date().toISOString(),
      usage_count: 1,
      note: ''
    };

    destinationMasterData.push(newDestinationRecord);
    saveDestinationToSheet(newDestinationRecord);

    return { id: newDestId, isNew: true, record: newDestinationRecord };
  }


  // =====================================================================
  // 🔍 หมวดการค้นหา (LOOKUP API) - สำหรับใช้ตอนดึง Lat/Long ไปใช้งานจริง
  // =====================================================================

  /**
   * 🎯 2. ค้นหาจุดส่งจาก "บุคคล" และ "สถานที่" (พระเอกของการดึงพิกัด)
   * ใช้เมื่อ "ตารางงานประจำวัน" รู้ทั้งคนและสถานที่
   */
  function findByPersonAndPlace(personId, placeId) {
    // หา Destination ทั้งหมดที่ตรงกับ คนๆนี้ และ สถานที่นี้
    const matches = destinationMasterData.filter(d => d.person_id === personId && d.place_id === placeId);
    
    if (matches.length === 0) return null;

    if (matches.length === 1) return matches[0];

    // THE ULTIMATE TRICK: ถ้าบังเอิญเจอหลายพิกัด (เช่น ลูกค้าสั่งไปไซต์งานเดิม แต่คนขับไปจอดรถกดพิกัดห่างกัน 100 เมตร จนระบบสร้าง Geo ใหม่)
    // ให้ระบบ "ฉลาดเลือก" โดยเอาพิกัดที่ "ไปส่งบ่อยที่สุด (usage_count สูงสุด)" กลับไปให้!
    matches.sort((a, b) => b.usage_count - a.usage_count);
    
    return matches[0]; // คืนค่าตัวที่แกร่งที่สุด
  }

  /**
   * 🎯 3. ดึงพิกัดทั้งหมดของ "สถานที่" นี้ (ไม่สนว่าใครเป็นคนรับ)
   * แก้ปัญหา: เปลี่ยนลูกน้องมารับของ แต่ไซต์งานเดิม
   */
  function getGeoIdsByPlace(placeId) {
    const matches = destinationMasterData.filter(d => d.place_id === placeId);
    // ดึงเฉพาะ Geo ID ออกมาแบบไม่ซ้ำกัน (Unique)
    const uniqueGeoIds = [...new Set(matches.map(d => d.geo_id))];
    return uniqueGeoIds;
  }

  /**
   * 🎯 4. ดึงพิกัดทั้งหมดของ "บุคคล" นี้
   * แก้ปัญหา: ลูกค้าสั่งประจำ แต่พิมพ์ที่อยู่มั่วมากจนหาไม่เจอ
   */
  function getGeoIdsByPerson(personId) {
    const matches = destinationMasterData.filter(d => d.person_id === personId);
    const uniqueGeoIds = [...new Set(matches.map(d => d.geo_id))];
    return uniqueGeoIds;
  }


  // --- Utility Functions ---
  function generateDestinationId() {
    return 'DST-' + Utilities.getUuid().substring(0, 8).toUpperCase();
  }

  function saveDestinationToSheet(record) {
    // ฟังก์ชันจำลอง: เรียก SpreadsheetApp เพื่อเพิ่มแถวใหม่ลง M_DESTINATION
  }

  function updateDestinationInSheet(record) {
    // ฟังก์ชันจำลอง: เรียก SpreadsheetApp เพื่ออัปเดต usage_count และ last_seen_date
  }

  return {
    loadData: loadData,
    linkDestination: linkDestination,
    findByPersonAndPlace: findByPersonAndPlace,
    getGeoIdsByPlace: getGeoIdsByPlace,
    getGeoIdsByPerson: getGeoIdsByPerson
  };

})();

    * `runDailyLookupProcess()`: สั่งรันระบบเพื่ออ่าน "ตารางงานประจำวัน" แล้วดึง Lat/Long กลับไปแปะ

* **`03_SetupSheets.gs` (Initialization)**

* **`14_Utils.gs` (Utility Tools)**

### 📥 กลุ่มที่ 2: ท่อส่งข้อมูลและซักล้าง (Intake & Cleansing)

    * `normalizeAddress(address)`: ล้างที่อยู่ ซ่อมคำผิด ลบที่อยู่เบิ้ลซ้ำซ้อน และขยายคำย่อ
    * `parseCoordinates(latlong)`: สกัดพิกัด ดักจับการพิมพ์สลับช่อง และกรองพิกัดที่อยู่นอกประเทศไทย
* **`16_GeoDictionaryBuilder.gs` (Thai Geography Base)**

### 🧠 กลุ่มที่ 3: สมองกลหลักและการผูกข้อมูล (The Core Brain & Masters)
หัวใจของโปรเจกต์ ที่ใช้แก้ปัญหาทั้ง 8 ข้อที่คุณต้องการ

* **`06_PersonService.gs` (Identity Management)**
    * `findOrInsertPerson()`: ค้นหาบุคคลจากเบอร์โทร -> ชื่อสะอาด ถ้าไม่เจอให้สร้าง ID ใหม่
    * `learnNewAlias()`: จดจำการพิมพ์ชื่อผิดๆ ถูกๆ ไว้ในตาราง `M_PERSON_ALIAS`
* **`07_PlaceService.gs` (Address Management)**
    * `findOrInsertPlace()`: ค้นหาสถานที่ ถ้าชื่อไม่ตรงแต่พิกัดเป๊ะ ให้ใช้ NLP วัดความคล้ายแล้วรวบเป็น ID เดียวกัน
    * `calculateTextSimilarity()`: อัลกอริทึมวัดความเหมือนของตัวอักษรแบบ Fuzzy Logic
* **`08_GeoService.gs` (Spatial Management)**
    * `findOrInsertGeoPoint()`: ค้นหาพิกัด หากห่างจากจุดเดิมไม่เกินรัศมี 50 เมตร ให้รวบเป็นจุดเดียวกัน
    * `calculateHaversineDistance()`: คำนวณระยะทางบนผิวโลก (ความโค้งของโลก) ระหว่าง 2 พิกัด
    * `parseGeoRemarks()`: แปลง Regex ในช่อง "หมายเหตุ" ของชีตภูมิภาค เพื่อใช้ตั้งค่าเงื่อนไขพิเศษ
* **`09_DestinationService.gs` (The Grand Synergy)**
    * `linkDestination()`: ผูก Person + Place + Geo เข้าด้วยกัน พร้อมนับยอดการส่ง (Usage Count)
    * `findByPersonAndPlace()`: ตามหาจุดส่งที่น่าเชื่อถือที่สุด (ไปบ่อยสุด) จากชื่อคนและสถานที่
* **`10_MatchEngine.gs` (The Orchestrator)**
    * `buildMasterData()`: ควบคุม Workflow ในการสูบข้อมูลจริงมาสร้าง Master Data (สอนระบบ)
    * `lookupLatLongForDailyTask()`: ควบคุม Workflow ในการเอา "ชื่อ+ที่อยู่มั่ว" มาตามล่าหาพิกัด Lat/Long กลับไปให้ระบบ

### 📊 กลุ่มที่ 4: ตรวจสอบและรายงานผล (Validation & Output)
