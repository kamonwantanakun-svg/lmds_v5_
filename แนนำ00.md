ใช้แบบ:

});

\#\#\# 🚨 1\. จุดผิดพลาดสำคัญ (Critical Bug): การใช้ \`\\\\n\` แทน \`\\n\`  
ใน Google Apps Script (JavaScript) การขึ้นบรรทัดใหม่ต้องใช้ \`\\n\` (Backslash ตัวเดียว) แต่ในโค้ดมีการเขียนเป็น \`\\\\n\` (Backslash 2 ตัว) ซึ่งจะทำให้แสดงผลออกมาเป็นตัวอักษร \`\\n\` จริงๆ บนหน้าจอแทนที่จะขึ้นบรรทัดใหม่

\*\*ตำแหน่งที่พบ:\*\*  
\*   \`runFullPipeline()\`: ในข้อความ \`ui.alert\` และ \`logInfo\`  
\*   \`openReviewQueue()\`: ในข้อความ \`ui.alert\`  
\*   \`checkSystemIntegrity()\`: การต่อข้อความ \`msg\`  
\*   \`setupEnvironment()\`: ในข้อความ \`ui.prompt\`  
\*   \`showVersionInfo()\`: การสร้างข้อความ \`msg\`

\*\*วิธีแก้ไข:\*\*  
เปลี่ยน \`\\\\n\` เป็น \`\\n\` ทั่วทั้งไฟล์

\*\*ตัวอย่าง:\*\*  
\`\`\`javascript  
// ❌ ผิด (จะแสดงคำว่า \\n ออกมา)  
ui.alert('✅ Full Pipeline สำเร็จ\!\\\\nใช้เวลา: ${elapsedSec} วินาที');

// ✅ ถูก (จะขึ้นบรรทัดใหม่)  
ui.alert(\`✅ Full Pipeline สำเร็จ\!\\nใช้เวลา: ${elapsedSec} วินาที\`);  
\`\`\`

\#\#\# 1\. ใช้ escape sequence \`\\\\n\` ผิดที่ ⇒ \*\*แสดงผล \`\\n\` เป็นตัวอักษรใน dialog\*\*

ปัญหาอยู่ที่ \`join('\\\\n')\` และ \`'\\\\n\\\\n'\` ใน \`checkSystemIntegrity()\`    
และ \`'\\\\n'\` ใน \`runFullPipeline()\` บางตำแหน่ง (แต่ใน \`runFullPipeline\` ใช้ถูกแล้วบางส่วน)

\*\*ตัวอย่างบรรทัดผิดใน \`checkSystemIntegrity\`:\*\*  
\`\`\`javascript  
msg \+= errors.map(e \=\> '  • ' \+ e).join('\\\\n') \+ '\\\\n\\\\n';  
\`\`\`  
\- \`'\\\\n'\` คือ literal 2 ตัวอักษร \= backslash \+ n    
\- ต้องการ newline จริง ๆ → ต้องใช้ \`'\\n'\`

\*\*แก้เป็น:\*\*  
\`\`\`javascript  
msg \+= errors.map(e \=\> '  • ' \+ e).join('\\n') \+ '\\n\\n';  
\`\`\`

ใน \`runFullPipeline\` ก็มี \`'\\\\n'\` ที่ผิดอีกที่หนึ่ง:  
\`\`\`javascript  
'จะรันกระบวนการทั้งหมด:\\\\n' \+   // อันนี้ถูก (ใช้ \\n)  
'  1\. โหลดข้อมูลดิบจากชีต SCGนครหลวงJWDภูมิภาค\\\\n' \+  // ตรงนี้ผิด \-\> ต้องเป็น \\n  
\`\`\`  
แต่สังเกตว่า \`'\\\\n'\` ต่อท้ายข้อความ จะทำให้แสดง \`\\n\` เป็นตัวหนังสือ    
\*\*ให้เปลี่ยนทุก \`\\\\n\` ที่ต้องการขึ้นบรรทัดใหม่เป็น \`\\n\` (ยกเว้นกรณีที่ต้องการ literal backslash จริง ๆ)\*\*

## **🔴 จุดที่ต้องแก้ไข (Bug จริง)**

### **1\. ปัญหา Escape Character `\\n` ในข้อความ Alert/Prompt**

**นี่คือบั๊กที่สำคัญที่สุดในไฟล์นี้ครับ** ในโค้ดของคุณใช้ `\\n` (double backslash \+ n) ในหลายจุด ซึ่งจะแสดงผลเป็นข้อความตัวอักษร `\n` ตรงๆ ใน Alert ของ Google Sheets ไม่ได้ขึ้นบรรทัดใหม่ตามที่ตั้งใจไว้

**ตำแหน่งที่ผิด** (มีอยู่หลายที่):

// ❌ ผิด \- ใน runFullPipeline()  
'จะรันกระบวนการทั้งหมด:\\\\n' \+  
'  1\. โหลดข้อมูลดิบจากชีต SCGนครหลวงJWDภูมิภาค\\\\n' \+

// ❌ ผิด \- ใน openReviewQueue()  
.alert('❌ ไม่พบชีต Q\_REVIEW\\\\nกรุณารัน "สร้างชีตทั้งหมด" ก่อน');

// ❌ ผิด \- ใน checkSystemIntegrity()  
ui.alert(\`✅ System Integrity: ปกติทุกอย่าง\!\\\\nVersion: ${APP\_VERSION}\`);  
msg \+= \`❌ พบ Error ${errors.length} รายการ:\\\\n\`;

// ❌ ผิด \- ใน setupEnvironment()  
'กรุณาใส่ Gemini API Key:\\\\n(ได้จาก https://aistudio.google.com/app/apikey)',

// ❌ ผิด \- ใน showVersionInfo()  
\`🚚 ${APP\_NAME}\\\\n\` \+

**วิธีแก้:** เปลี่ยน `\\n` เป็น `\n` ทุกจุด

// ✅ ถูกต้อง  
'จะรันกระบวนการทั้งหมด:\\n' \+  
'  1\. โหลดข้อมูลดิบจากชีต SCGนครหลวงJWDภูมิภาค\\n' \+

💡 **หมายเหตุ:** ใน JavaScript ปกติ `\n` คือ newline character แล้ว ส่วน `\\n` คือการ escape เพื่อให้ได้ตัวอักษร backslash \+ n ตรงๆ (มักใช้ใน JSON string หรือ regex) ในกรณีนี้คุณต้องการขึ้นบรรทัดใหม่จริงๆ จึงต้องใช้ `\n` ตัวเดียว
